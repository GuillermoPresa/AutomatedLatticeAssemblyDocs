\hypertarget{class_illa_controller_v3_autotoc_md8}{}\doxyparagraph{Gearing ratio}\label{class_illa_controller_v3_autotoc_md8}
The configuration also stores the gearing ratio of the motors in the joints. This is used to compute the joint angles from the motor angles and viceversa. This ratio is defined as the number of rotations the motor performes per rotation of the joint. Meaning that a ratio of 2 means that for every rotation of the joint the motor will rotate 1 rotations meaning the joint torque is enhanced. The gearing ratio is stored in the Differential\+Joint and Single\+Joint condigurations.

\hypertarget{class_illa_controller_v3_autotoc_md12}{}\doxyparagraph{Enabling the servo motors\+:}\label{class_illa_controller_v3_autotoc_md12}
Before the servo motors are enabled, the motor microcontroller is active and reporting its position but compliant with the motor unpowered. At this point the target configuration is being updated to the current configuration as moved by the user, in order to prevent a big displacement when connection is stablished.

Once connection is stablished the motors become fully active. The target configuration is no longer updated to match the current configuration but rather it can be updated via the user interface leading to robot movements to match the current position to the target position.

Compute the possible movements for a robot from a given configuration.

This function computes the possible movements for a robot from a given configuration, where the configuration is specified by a \mbox{\hyperlink{class_robot_configuration}{Robot\+Configuration}} object (Configuration) and includes the robot\textquotesingle{}s current position, orientation, and joint angles. The function returns a list of possible movements in the form of \mbox{\hyperlink{class_movement}{Movement}} objects, each of which represents a possible sequence of joint angles and end effector positions that the robot can use to move to a specific target node in the lattice.

If the Check\+Collisions flag is set to true, the function will check for collisions between the robot and any obstacles in the environment during the movements. If a collision is detected, the function will eliminate the movement from the list of possible movements.
