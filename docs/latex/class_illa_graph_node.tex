\doxysubsection{Detailed Description}
Navigation graph for ILLA inchowrm locomotion lattice assemblers that employ inverse kinnematics and dont have preprogramed movements.

The navigation graph is directional, nmeaning that neighbours are not necesarily symmetric. For this reason there exists both\+: ~\newline
 $\ast$\+\_\+neighbours\+: $\ast$ Stores the neoigbour of a node. (outgoing edge) ~\newline
$\ast$\+\_\+incoming\+Neighbours\+: $\ast$ Stores what nodes have the current node as neighbour (incoming edge) ~\newline
 This allows traversal of the graph using fordward and bidirectional search instead on just backwards without a large computational headroom (at the cost of additional memory) ~\newline
 ~\newline
~\newline
 The edge weight depends on the initial conditions, meaning that no predefined cost can be stablished and that each edge´s cost depends on the robot´s history. ~\newline
 In order to account for this during path planning and be able to find the shortest path regardless of initial conditions the following procedure is utilized. ~\newline
 An example is presented below\+: ~\newline
 Here a 3-\/1-\/1 node configuration graph can be seen. The blue states represent the dead states. The green are the alive states. and the grey is the unvisited state. We are investigating the fastest way to reach the goal (g) state.~\newline
 It must be noted that depending on the route the robot takes to reach to node c, the joint configuration in that node will be different, thus, the cost of the c-\/\texorpdfstring{$>$}{>}g edge is different according to the robot´s hystogram. ~\newline
 This is taken into accound in the path planning. And this is only further complicated by the existance of more incomming neighbours to the goal state.



~\newline
 ~\newline


The \+\_\+incoming\+Neighbour\+Cost\+Dictionary, stores for incomming neighbours, the previous node, inital \mbox{\hyperlink{class_pos_j}{PosJ}} and the cost to reach the goal node from the origin. ~\newline
 The incomming neighbours previous node, initial \mbox{\hyperlink{class_pos_j}{PosJ}} and cost to reach are stored in the form of a \mbox{\hyperlink{struct_illa_graph_node_1_1_cost_struct}{Cost\+Struct}}

The structu Pos\+JAnd\+Cost is utilized to store this information\+: ~\newline
 Compute\+Cost\+To\+Reach\+From\+Origin populates this dictionary and structs and returns the path with the minimal cost.

\begin{DoxyWarning}{Warning}
$\ast$\+Starting node has to be assigned an incomming cost struct with itself as the previous incomming node. This way the search algorithm can compute its initial cost. $\ast$ In essence the definition of the initial node can be that of a node whose previous node is itself (in a movement of zero cost). 
\end{DoxyWarning}
