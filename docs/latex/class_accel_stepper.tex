\doxysubsection{Detailed Description}
Support for stepper motors with acceleration etc. 

This defines a single 2 or 4 pin stepper motor, or stepper moter with fdriver chip, with optional acceleration, deceleration, absolute positioning commands etc. Multiple simultaneous steppers are supported, all moving at different speeds and accelerations.

\begin{DoxyParagraph}{Operation}
This module operates by computing a step time in microseconds. The step time is recomputed after each step and after speed and acceleration parameters are changed by the caller. The time of each step is recorded in microseconds. The \mbox{\hyperlink{class_accel_stepper_a608b2395b64ac15451d16d0371fe13ce}{run()}} function steps the motor once if a new step is due. The \mbox{\hyperlink{class_accel_stepper_a608b2395b64ac15451d16d0371fe13ce}{run()}} function must be called frequently until the motor is in the desired position, after which time \mbox{\hyperlink{class_accel_stepper_a608b2395b64ac15451d16d0371fe13ce}{run()}} will do nothing.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Positioning}
Positions are specified by a signed long integer. At construction time, the current position of the motor is consider to be 0. Positive positions are clockwise from the initial position; negative positions are anticlockwise. The current position can be altered for instance after initialization positioning.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Caveats}
This is an open loop controller\+: If the motor stalls or is oversped, \mbox{\hyperlink{class_accel_stepper}{Accel\+Stepper}} will not have a correct idea of where the motor really is (since there is no feedback of the motor\textquotesingle{}s real position. We only know where we {\itshape think} it is, relative to the initial starting point).
\end{DoxyParagraph}
\begin{DoxyParagraph}{Performance}
The fastest motor speed that can be reliably supported is about 4000 steps per second at a clock frequency of 16 MHz on Arduino such as Uno etc. Faster processors can support faster stepping speeds. However, any speed less than that down to very slow speeds (much less than one per second) are also supported, provided the \mbox{\hyperlink{class_accel_stepper_a608b2395b64ac15451d16d0371fe13ce}{run()}} function is called frequently enough to step the motor whenever required for the speed set. Calling \mbox{\hyperlink{class_accel_stepper_adfb19e3cd2a028a1fe78131787604fd1}{set\+Acceleration()}} is expensive, since it requires a square root to be calculated.
\end{DoxyParagraph}
Gregor Christandl reports that with an Arduino Due and a simple test program, he measured 43163 steps per second using \mbox{\hyperlink{class_accel_stepper_aa4a6bdf99f698284faaeb5542b0b7514}{run\+Speed()}}, and 16214 steps per second using \mbox{\hyperlink{class_accel_stepper_a608b2395b64ac15451d16d0371fe13ce}{run()}};

Constructor. You can have multiple simultaneous steppers, all moving at different speeds and accelerations, provided you call their \mbox{\hyperlink{class_accel_stepper_a608b2395b64ac15451d16d0371fe13ce}{run()}} functions at frequent enough intervals. Current Position is set to 0, target position is set to 0. Max\+Speed and Acceleration default to 1.\+0. The motor pins will be initialised to OUTPUT mode during the constructor by a call to \mbox{\hyperlink{class_accel_stepper_aa279a50d30d0413f570c692cff071643}{enable\+Outputs()}}. 
\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em interface} & Number of pins to interface to. Integer values are supported, but it is preferred to use the \mbox{\hyperlink{class_accel_stepper_a73bdecf1273d98d8c5fbcb764cabeea5}{Motor\+Interface\+Type}} symbolic names. \mbox{\hyperlink{class_accel_stepper_a73bdecf1273d98d8c5fbcb764cabeea5ac3523e4cf6763ba518d16fec3708ef23}{Accel\+Stepper\+::\+DRIVER}} (1) means a stepper driver (with Step and Direction pins). If an enable line is also needed, call \mbox{\hyperlink{class_accel_stepper_a56a81c5f00d02ca19646718e88e974c0}{set\+Enable\+Pin()}} after construction.
