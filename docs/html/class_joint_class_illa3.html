Assembly definition

The assembly constraint of the joint is defined by the following properties:

assemblyPosition : Vector3 with (x,y,z) assembly constraint of the current node wrt base joint
assemblyRotation : Vector3 with euler angles (x,y,z) of the assembly constraint of the current node wrt base joint.
The assembly constraints are provided in Hierarchy agnosntic, Absolute frame. This is more convinient for the user when defining a robot as it allows to build the robot within Unity prefab editor and copy paste the assembly constraints.

Frame definition

The frame of the joint (position and orientation) can be defined with respect to the base joint (aboslute) or previous joint (relative)

Relative frame: The joint is defined in a relative frame, with respect to the previous joint.
Used in: hARelativeAssemblyPosition, hARelativeAssemblyRotation, hDRelativeAssemblyPosition, hDRelativeAssemblyRotation

Absolute frame: The joint is defined in a relative frame, with respect to the base joint.
Used in: hDAbsoluteAssemblyRotation, hDAbsoluteAssemblyPosition, hDCurrentOrientation??, hDCurrentPosition??

Hierarchy dependence

When the robot hierarchy is inverted, the next node and previous node to control will be different than that of the assembly definition.

Hierarchy agnostic (HA): Hierachy agnostic properties are those that are not affected by the hierarchy inversion such that the next and previous node are always the same and dont change, always representing the assembly constraints.
Used in: hARelativeAssemblyPosition, hARelativeAssemblyRotation

Hierarchy dependent (HD): Hierarchy dependent properties are those that are affected by the the hierarchy inversion such that the next and previous node swap after a hierarchy inversion.
Used in: hDRelativeAssemblyPosition, hDRelativeAssemblyRotation, hDAbsoluteAssemblyRotation, hDAbsoluteAssemblyPosition

◆ InvertHierarchy()

void JointClassIlla3.InvertHierarchy ( )

Inverts the hierarchy of the joint. This entails two parts:

1) The previous joint and next joint are swapped around. 2) The current assembly rotation its updated to it's correct value.

Additionally the IsInverted flag is set to the new status.
