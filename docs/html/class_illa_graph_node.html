Detailed Description

Navigation graph for ILLA inchowrm locomotion lattice assemblers that employ inverse kinnematics and dont have preprogramed movements.

The navigation graph is directional, nmeaning that neighbours are not necesarily symmetric. For this reason there exists both:
*_neighbours: * Stores the neoigbour of a node. (outgoing edge)
*_incomingNeighbours: * Stores what nodes have the current node as neighbour (incoming edge)
This allows traversal of the graph using fordward and bidirectional search instead on just backwards without a large computational headroom (at the cost of additional memory)


The edge weight depends on the initial conditions, meaning that no predefined cost can be stablished and that each edge´s cost depends on the robot´s history.
In order to account for this during path planning and be able to find the shortest path regardless of initial conditions the following procedure is utilized.
An example is presented below:
Here a 3-1-1 node configuration graph can be seen. The blue states represent the dead states. The green are the alive states. and the grey is the unvisited state. We are investigating the fastest way to reach the goal (g) state.
It must be noted that depending on the route the robot takes to reach to node c, the joint configuration in that node will be different, thus, the cost of the c->g edge is different according to the robot´s hystogram.
This is taken into accound in the path planning. And this is only further complicated by the existance of more incomming neighbours to the goal state.

3-1-1 Node Graph



The _incomingNeighbourCostDictionary, stores for incomming neighbours, the previous node, inital PosJ and the cost to reach the goal node from the origin.
The incomming neighbours previous node, initial PosJ and cost to reach are stored in the form of a CostStruct

The structu PosJAndCost is utilized to store this information:
ComputeCostToReachFromOrigin populates this dictionary and structs and returns the path with the minimal cost.

Warning
*Starting node has to be assigned an incomming cost struct with itself as the previous incomming node. This way the search algorithm can compute its initial cost. * In essence the definition of the initial node can be that of a node whose previous node is itself (in a movement of zero cost).
