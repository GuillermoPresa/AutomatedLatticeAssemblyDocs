Planning on the graph:

 Planning is the reasoning side of acting.

Utilizing the dynamic programming principle based on Bellman's principle of optimality:

 An optimal policy has the property that whatever the initial state and initial decision are, the remaining decisions must constitute an optimal policy with regard to the
 state resulting from the first decision.

Process:

At any point in the search, we can classify the states into 3 groups; Unvisited, dead or alive
Unvisited: states that have not been visited yet
Dead: States that have been visited togeather with every possible next state
Alive: State that has been visited but whose possible next states have yet to be visited
Notation:
States are denoted by the letter X.

X: states
X_goal
X_initial

Search algorithms:
f(x) = g(x) + e h(x)
f(x): Cost utilized for evaluated the priority of the priority queue. Deternubes which nodes will be visited next.
g(x): Cost of the path from the starting state x_{i} to the current state_{x}
h(x): Heuristic function that aproximates the cost from the current state x to the goal state x_{g}
e: Admissibility criterion

◆ Heatmap()

void IllaGraph.Heatmap ( )

Generates a heatmap on the nodes of the graph based on their cost to reach from starting node.

This function generates a heatmap visualization on the nodes of a graph by assigning a color to each node based on its cost to reach from the starting node.

It uses the VisitedNodes list to filter out the visited nodes of the graph, and then sorts the nodes based on their cost. After that, it assigns a heatmap color to each node based on its cost by using the getHeatmapColorByValue function from the UITools class. Finally, it overrides the color of the node's parent GameObject with the generated heatmap color, and adds the heatmap texture to the rendering job queue of the referenceScriptReference.

Parameters
None.
Returns
None.
Note
The graph must be initialized with nodes and edges before calling this function.
