◆ ReferenceScriptReference

References RobotCoordinator.ReferenceScriptReference

Efficient construction of digital structures can employ a variety of assemblers and robotic assets. A work package system has been integrated into the construction flow to connect high-level goals and detailed implementation level control of different robotic assets.

Work-packages are groups of tasks.

A task is a general representation of work that can be budgeted and executed by different assembler types. An example of a task is moving to a different far-away node.

Robot controllers break down tasks into actions to perform them. Actions are the minimum unit of work that can be executed, these vary per assembler type. Following the previous example, when a task is moving to a far-away node, the actions would be multiple PosJ movements and screwing actions for an inchworm locomotion robot or a flight hop for a drone assembler.
