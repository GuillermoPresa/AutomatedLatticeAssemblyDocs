Detailed Description

Main governing script of 5DOF ILLA inchworm locomotion lattice assemblers.

Flow

1) ILLAV3 is initialized 2) Configuration file is loaded 3) Serial connection is stablished

State machine

ILLAV3 is a state machine. An introduction to finate state machines can be found in YT video State machine This state machine implementation relies on general entering and exiting state mechanisms rather than custom state to state switching that is different for every transition.

States

The following states can be found

  • Spawned : ILLAV3 class was just initialized and nothing has been done to it
  • Configured : A configuration has be assigned to the robot
  • Connected : (Also called Waiting) A serial communication is established with physical robot. Default state when the robot is not performing tasks but is connected and ready

  • E-Stop : State used when robot is stopped in an emergency stop situation.

Configuration

The parameters of the robot are saved on the ILLA configuration file. The joint and servo configuration are saved in the form of two differential joint configurations that embody the config of their corresponding servos and joints as well as the configuration of a lonely servo and join for the middle axis. By convention, as it is deeply rooted in the base of the ILLA controller, the configuration is loaded

Gearing ratio

The configuration also stores the gearing ratio of the motors in the joints. This is used to compute the joint angles from the motor angles and viceversa. This ratio is defined as the number of rotations the motor performes per rotation of the joint. Meaning that a ratio of 2 means that for every rotation of the joint the motor will rotate 1 rotations meaning the joint torque is enhanced. The gearing ratio is stored in the DifferentialJoint and SingleJoint condigurations.

Enabling the servo motors:

Before the servo motors are enabled, the motor microcontroller is active and reporting its position but compliant with the motor unpowered. At this point the target configuration is being updated to the current configuration as moved by the user, in order to prevent a big displacement when connection is stablished.

Once connection is stablished the motors become fully active. The target configuration is no longer updated to match the current configuration but rather it can be updated via the user interface leading to robot movements to match the current position to the target position.

#Controller board and comunication.

The controller on board the ILLAV3 is ILLA mainboard, an adapter board that connects the lynxmotion bus to an Xbee module for wireless comunication and an arduino nano every for USB and on board instructions

Wireless communication.

For wireless comunication a 2.4Ghz radio module is used. For its ease of use and realibility and ease of integration in the lynxmotion ecosystem via the LSS-ADA board, Xbee modules were chosen. The Xbee 3 Zigbee module is used

An Through-Hole mounting XBee 3 Zigbee (2.4GHz) module is employed. In particular the U.FL antenna version (XB3-24Z8UT-J) is used on the robot and a RP-SMA antenna version (XB3-24Z8UT-J ) is used on the computer side allowing for larger antennas.

Xbee configuration

Computes the Inverse Kinematics (IK) solutions for an ILLA 5DOF robotic arm.

This function aims to find joint configurations that position the end effector of the arm at the given target position and orientation. Given the limitations of the 5DOF arm, the function implements mechanisms to ensure the correct IK solutions by clamping the target position into the solution space of the arm.

Parameters
targetPosXThe desired end PosX (position and orientation) for the robot's end effector.
inputToleranceAngular input tolerance. If the input target position deviates more than the tolerance from the valid solution space an error will be raised. Default is 10 degrees.
correctInputIf set to true, the function will make corrections to the input to ensure valid solutions. Default is true.
debugGizmosIf set to true, debug gizmos will be activated. Default is true.
Returns
Returns a Movement object containing all possible solutions for reaching the desired position.
Warning
This method is designed for a 5DOF ILLA robot. It considers limitations in the robot's solution space. Invalid solutions (those that are outside the robot's capabilities) may be discarded.
Exceptions
Throwsan Exception if the angle tolerance is surpassed in the IK input.

◆ InvertHierarchyAllJoints()

void IllaControllerV3.InvertHierarchyAllJoints ( )

Inverts the hierarchy of all joints in the robotic arm.

This method first calls the InvertHierarchy() method on each of the five joints, reversing the order in which they are connected to one another. The method then sets the IsBase property of the first and last joints to false, indicating that they are not the base of the arm.

Next, the method saves the current target angles for each joint in temporary variables, and then flips the IsInverted flag. If the flag is now true, the method sets the BaseJoint, ShoulderJoint, WristJoint, and GripperJoint variables to the last, third-to-last, second-to-last, and first joints, respectively. It also sets the corresponding JointScript variables to the script components attached to those joints. The method then disables the "Strut1NoInvert" and "Strut2NoInvert" game objects, and enables the "Strut1Inverted" and "Strut2Inverted" game objects.

If the IsInverted flag is not set (i.e., it is false), the method sets the BaseJoint, ShoulderJoint, WristJoint, and GripperJoint variables to the first, second, fourth, and fifth joints, respectively. It also sets the corresponding JointScript variables to the script components attached to those joints. The method then enables the "Strut1NoInvert" and "Strut2NoInvert" game objects, and disables the "Strut1Inverted" and "Strut2Inverted" game objects.

Finally, the method calls the MakeBase() method on the BaseJointScript variable, which sets the base joint as the root of the robotic arm hierarchy.

Compute the possible movements for a robot from a given configuration.

This function computes the possible movements for a robot from a given configuration, where the configuration is specified by a RobotConfiguration object (Configuration) and includes the robot's current position, orientation, and joint angles. The function returns a list of possible movements in the form of Movement objects, each of which represents a possible sequence of joint angles and end effector positions that the robot can use to move to a specific target node in the lattice.

If the CheckCollisions flag is set to true, the function will check for collisions between the robot and any obstacles in the environment during the movements. If a collision is detected, the function will eliminate the movement from the list of possible movements.

Parameters
ConfigurationThe robot's current configuration.
CheckCollisionsA flag indicating whether to check for collisions during the movements.
Returns
A list of possible movements for the robot.
