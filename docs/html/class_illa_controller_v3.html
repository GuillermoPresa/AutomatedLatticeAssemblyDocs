Detailed Description

Main governing script of ILLA inchworm locomotion lattice assemblers.

Note
Integrates the concepts from PosJ, PosX, RobotConfiguration and Movement.
Warning
Illa is based on Inverse Kinnematics and high flexibility. It has the capability of performing collision estimation. Some of these calls can however be very computationally expensive.
Todo:

Delete collision detection scenee class and related content in the inchworm coontroller

Transition I.K from custom implementation to a simpler to understand I.K solution with the robot defined according to Denavit Hartenberg parameters (instead on my arbitrary definitions) as showcased in the Iliukhin's paper mentioned in the I.K function. Additionally this opens up the possibility of cleaning the I.K and joint position from bodges.

Remove joint script and save the target angle in the denavit hartemberg matrix or in an array in the controller.

Play with robot reach radius to optimize lattice discovery by minimizing unreachable discoveries.

Refine public and private variable status

◆ FrameMapNavigationSpace()

IEnumerator IllaControllerV3.FrameMapNavigationSpace ( RobotConfiguration  StartingConfiguration)

This function generates a graph representing a navigation space for a robot starting from the provided configuration.
The graph is generated using a breadth-first search approach with non-deepening steps.
The function uses a stopwatch to ensure the time elapsed in each frame is less than a specified maximum thread locking time.
The function yields control after each frame to prevent the function from taking too long to execute.

Parameters
StartingConfigurationThe starting configuration of the robot.
Returns
An IEnumerator object that generates a graph representing a navigation space for a robot.

◆ InvertHierarchyAllJoints()

void IllaControllerV3.InvertHierarchyAllJoints ( )

Inverts the hierarchy of all joints in the robotic arm.

This method first calls the InvertHierarchy() method on each of the five joints, reversing the order in which they are connected to one another. The method then sets the IsBase property of the first and last joints to false, indicating that they are not the base of the arm.

Next, the method saves the current target angles for each joint in temporary variables, and then flips the IsInverted flag. If the flag is now true, the method sets the BaseJoint, ShoulderJoint, WristJoint, and GripperJoint variables to the last, third-to-last, second-to-last, and first joints, respectively. It also sets the corresponding JointScript variables to the script components attached to those joints. The method then disables the "Strut1NoInvert" and "Strut2NoInvert" game objects, and enables the "Strut1Inverted" and "Strut2Inverted" game objects.

If the IsInverted flag is not set (i.e., it is false), the method sets the BaseJoint, ShoulderJoint, WristJoint, and GripperJoint variables to the first, second, fourth, and fifth joints, respectively. It also sets the corresponding JointScript variables to the script components attached to those joints. The method then enables the "Strut1NoInvert" and "Strut2NoInvert" game objects, and disables the "Strut1Inverted" and "Strut2Inverted" game objects.

Finally, the method calls the MakeBase() method on the BaseJointScript variable, which sets the base joint as the root of the robotic arm hierarchy.

◆ NonDeepeningMapNavSpace()

List< RobotConfiguration > IllaControllerV3.NonDeepeningMapNavSpace ( RobotConfiguration  StartingConfiguration)

NonDeepeningMapNavSpace - Generates a list of RobotConfigurations for all possible movements from the StartingConfiguration within the NavigationMesh.

Parameters
StartingConfigurationThe RobotConfiguration to start the navigation from.
Returns
List<RobotConfiguration> A list of RobotConfigurations for all possible movements from the StartingConfiguration within the NavigationMesh.

This function generates a list of RobotConfigurations for all possible movements from the StartingConfiguration within the NavigationMesh.
The NavigationMesh keeps track of visited nodes to avoid visiting the same node multiple times. If a node has already been visited, the function returns an empty list.
The function first gets all possible movements from the StartingConfiguration, then adds nodes, edges, and RobotConfigurations to the NavigationMesh and ToReturnList respectively if they are not already in the NavigationMesh and ToReturnList.
The function also updates the number of times the StartingConfiguration's BaseNode has been visited and moved to.

Note
This function assumes that the NavigationMesh has already been initialized.

◆ PossibleMovements()

List< Movement > IllaControllerV3.PossibleMovements ( RobotConfiguration  Configuration,
bool  CheckCollisions = true 
)

Compute the possible movements for a robot from a given configuration.

This function computes the possible movements for a robot from a given configuration, where the configuration is specified by a RobotConfiguration object (Configuration) and includes the robot's current position, orientation, and joint angles. The function returns a list of possible movements in the form of Movement objects, each of which represents a possible sequence of joint angles and end effector positions that the robot can use to move to a specific target node in the lattice.

If the CheckCollisions flag is set to true, the function will check for collisions between the robot and any obstacles in the environment during the movements. If a collision is detected, the function will eliminate the movement from the list of possible movements.

Parameters
ConfigurationThe robot's current configuration.
CheckCollisionsA flag indicating whether to check for collisions during the movements.
Returns
A list of possible movements for the robot.
