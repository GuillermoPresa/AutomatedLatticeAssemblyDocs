Detailed Description

Main governing script of ILLA inchworm locomotion lattice assemblers.

Note
Integrates the concepts from PosJ, PosX, RobotConfiguration and Movement.
Warning
Illa is based on Inverse Kinnematics and high flexibility. It has the capability of performing collision estimation. Some of these calls can however be very computationally expensive.

◆ FwdKinDHParamsGlobF()

Vector3 InchwormRobotController.FwdKinDHParamsGlobF ( PosJ  KinConfiguration)

Computes the forward kinematics using the Denavit-Hartenberg parameters and global frame

Parameters
KinConfiguration- The joint angle configuration
Returns
Vector3 - The position of the gripper in the global frame The function takes a joint angle configuration as an input and computes the forward kinematics using the Denavit-Hartenberg parameters. The joint angles are temporarily updated to the given configuration, the position of the gripper is calculated and then the joint angles are returned to their original configuration. The result is the position of the gripper in the global frame, represented as a Vector3 object. If the computation of the forward kinematics fails, an error message is displayed.

◆ InvertHierarchyAllJoints()

void InchwormRobotController.InvertHierarchyAllJoints ( )

Inverts the hierarchy of all joints in the robotic arm.

This method first calls the InvertHierarchy() method on each of the five joints, reversing the order in which they are connected to one another. The method then sets the IsBase property of the first and last joints to false, indicating that they are not the base of the arm.

Next, the method saves the current target angles for each joint in temporary variables, and then flips the IsInverted flag. If the flag is now true, the method sets the BaseJoint, ShoulderJoint, WristJoint, and GripperJoint variables to the last, third-to-last, second-to-last, and first joints, respectively. It also sets the corresponding JointScript variables to the script components attached to those joints. The method then disables the "Strut1NoInvert" and "Strut2NoInvert" game objects, and enables the "Strut1Inverted" and "Strut2Inverted" game objects.

If the IsInverted flag is not set (i.e., it is false), the method sets the BaseJoint, ShoulderJoint, WristJoint, and GripperJoint variables to the first, second, fourth, and fifth joints, respectively. It also sets the corresponding JointScript variables to the script components attached to those joints. The method then enables the "Strut1NoInvert" and "Strut2NoInvert" game objects, and disables the "Strut1Inverted" and "Strut2Inverted" game objects.

Finally, the method calls the MakeBase() method on the BaseJointScript variable, which sets the base joint as the root of the robotic arm hierarchy.

◆ PossibleMovements() [2/2]

List< Movement > InchwormRobotController.PossibleMovements ( RobotConfiguration  Configuration,
bool  CheckCollisions 
)

Compute the possible movements for a robot from a given configuration.

This function computes the possible movements for a robot from a given configuration, where the configuration is specified by a RobotConfiguration object (Configuration) and includes the robot's current position, orientation, and joint angles. The function returns a list of possible movements in the form of Movement objects, each of which represents a possible sequence of joint angles and end effector positions that the robot can use to move to a specific target node in the lattice.

If the CheckCollisions flag is set to true, the function will check for collisions between the robot and any obstacles in the environment during the movements. If a collision is detected, the function will eliminate the movement from the list of possible movements.

Parameters
ConfigurationThe robot's current configuration.
CheckCollisionsA flag indicating whether to check for collisions during the movements.
Returns
A list of possible movements for the robot.

◆ ReachTargetNode()

Movement InchwormRobotController.ReachTargetNode ( GameObject  TargetNode,
bool  debugGizmos = true 
)

Provides the Inverse Kinnematics (I.K) pose solution.

The flag debugGizmos is used to show in screen vectors important for the I.K solution

Magenta: Solution space plane nomrmal vector White: Rotationless wrist (J4 non inv) pointing vector (must be in the plane defined by the magneta vector) Black: Rotation included wrist pointing vector (must be in the plane defined by the magneta vector). The difference between the white and black vectors represents the rotation of the wrist.

Returns a movement (Starting config + List<PosJ>) detailing how to reach the target node.

Warning
Does not validate the Movement solutions for collisions
